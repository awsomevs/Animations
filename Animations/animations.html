Defining Complex, Multi-Step Motion with CSS.

The Core Concept

CSS Animations use **Keyframes** to control complex motion. Keyframes define the element's appearance (CSS properties) at specific points (waypoints) along the animation timeline. The browser smoothly interpolates the steps between these keyframes.

How CSS Animations Work

Keyframe Syntax: From/To Example

The **`@keyframes`** rule defines the sequence using percentages ($0\%$ to $100\%$) or the keywords **`from`** ($0\%$) and **`to`** ($100\%$).

@keyframes slide-in {
    /* from is 0% */
    from { 
        transform: translateX(-150%); 
        opacity: 0;
    } 
    /* to is 100% */
    to { 
        transform: translateX(0); 
        opacity: 1;
    } 
}

Applying the Animation:

.animated-element {
    animation-name: slide-in;
    animation-duration: 1.5s;
    animation-iteration-count: 1; 
    animation-fill-mode: forwards;
}

All CSS Animation Properties

These eight longhand properties provide complete control over the animation's playback:

Property Description
animation-name Links the element to the name defined in the @keyframes rule.
animation-duration Sets the time one full cycle of the animation takes (e.g., 2s or 500ms).
animation-timing-function Defines the acceleration curve (the speed) of the animation (e.g., linear, ease-in-out).
animation-delay Specifies the delay before the animation starts playing.
animation-iteration-count Determines how many times the animation should loop (e.g., 3 or infinite).
animation-direction Sets whether the animation plays forward (normal) or alternates between forward and backward cycles (alternate).
animation-fill-mode Controls the styles applied to the element before the animation starts and after it finishes (e.g., forwards keeps the final style).
animation-play-state Allows the animation to be paused or resumed (paused or running).

The **animation** property is a **shorthand** for setting all these properties in one line.
